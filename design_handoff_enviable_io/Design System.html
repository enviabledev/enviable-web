Design System

Building blocks reused across every Enviable I&O screen. Reference this page when designing new flows; never invent components outside this set without flagging it.
v1.0 · 18 May 2026
01

Buttons

Four variants × three sizes. Primary for the main action on a view (one per region). Secondary for non-primary actions. Tertiary for inline/dense placements. Destructive reads red and always lives behind a confirmation.

The workhorse component. Sticky header, sortable column, zebra rows, row hover and selection, a filter toolbar above, an overflow-menu actions column, and pagination at the bottom. Numeric cells are right-aligned with tabular figures. IDs use the mono face.

Breadcrumbs above the title, page title at 24px, optional one-line description below, and primary actions right-aligned. Used as the first block in the content area of every screen.

07

Modal dialog

Medium width (520px). Header with title and close button; body for the form/content; footer right-aligns the secondary "Cancel" and primary action. Reserve modals for destructive confirms, focused single-task forms, and "are you sure" checkpoints.

08

Toast notifications

Bottom-right stack. Coloured left bar indicates type. Auto-dismiss after ~5 seconds (errors are sticky). Title + one-line message; never reflow the page to make room.

09

Empty state

Plain panel — no decorative illustration. A subtle circular glyph, a heading, a one-line explanation, a primary action. Used inside any container that has zero rows / records / items.

No purchase orders yet

When a procurement officer raises a PO, it'll show up here. Start by issuing one to a registered supplier.

10

Section card

White card with a 3px coloured left border. Use the colour to communicate the semantic stake of the group (navy = neutral / brand, green = confirmed, amber = needs attention, red = problem). Group related fields, totals, or summary blocks inside.
